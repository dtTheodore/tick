# Pricing engine — TypeScript port

This directory mirrors `games/tap-trading/backend/pricing-engine/`
on a function-by-function basis. The Rust crate is the canonical
implementation; this port exists so the UI can compute per-cell
multipliers at 10 Hz without a network round trip.

**Tolerance contract:** for any input the backend would accept,
`compute_multiplier` here returns a value within **1e-6 absolute**
of the Rust `compute_multiplier`. The TS-vs-Rust parity is enforced
by `__tests__/parity.test.ts`, which consumes fixtures generated by
`pricing-engine/src/bin/gen_fixtures.rs`.

**Editing rules:**

- If you change a formula here, change it in Rust too, regenerate
  fixtures, and re-run parity tests.
- The math primitives `Math.exp/log/pow/sin` may differ from Rust's
  `f64::exp/ln/powf/sin` by ≤1 ULP. The drift gate is 3%, so this
  is safe in practice. If parity fails by more than 1e-6, the bug
  is in this port, not in the primitive.
- Do not optimize. Readability beats microseconds.
